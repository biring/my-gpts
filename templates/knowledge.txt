Delete this block when generating a knowledge file.

Scope: this template is for ONE knowledge file that supports ONE instructions.txt. A knowledge file holds domain rules, examples, and edge cases that would push the Instructions field over its character limit. It explains and exemplifies; it does not replace or restate the Instructions field's authority.

Core sections [CORE] apply to every knowledge file. Optional sections [OPTIONAL] apply only in specific situations (noted inline); delete the section if it does not apply.

SECTION ORDER AND WHY:
  1. PURPOSE                               [CORE]     - context: what, why
  2. SCOPE                                 [CORE]     - context: when
  3. RULES                                 [CORE]     - the requirements
  4. NAMING CONVENTIONS                    [OPTIONAL] - a specific sub-rule
  5. FORBIDDEN PRACTICES                   [CORE]     - the anti-rules
  6. DECISION CRITERIA FOR AMBIGUOUS CASES [OPTIONAL] - resolving judgment calls
  7. NO-OP / STABILITY CRITERIA            [OPTIONAL] - gate before generating
  8. EXAMPLE 1+                            [CORE]     - what compliant output looks like
  9. CHECKLIST                             [CORE]     - final self-check

FORMATTING HIERARCHY (plain text, no markdown):
  SECTION NAME     — top level, ALL CAPS, no prefix
  Label:           — sub-label, title case + colon
  - item           — list item under a label or section
  # Subsection:    — named subsection; use only when a section has three or more named groups; one or two groups use plain Label: instead
  - item
  <note>           — authoring instruction or hint; remove entirely when generating a knowledge file

SPACING RULES:
  2 blank lines before every SECTION NAME
  No blank line between a section heading and its first content
  1 blank line before each # subsection
  No blank line between a Label: or # Subsection: and its dash items
  1 blank line between plain Labels: within the same section

EXAMPLE may be numbered (EXAMPLE 1, EXAMPLE 2, ...) and repeated as needed. Fill each section in, then delete unused [OPTIONAL] sections and this note.
