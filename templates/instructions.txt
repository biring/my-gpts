=== TEMPLATE NOTES =================================================

Delete this block when generating GPT instructions

Single-purpose GPT: one set of inputs per run, one output with optional report.

CORE sections are required in every GPT.
OPTIONAL sections: include only when the noted condition applies; delete
the section otherwise.

FORMATTING HIERARCHY (plain text, no markdown):
  SECTION NAME     — top level, ALL CAPS, no prefix
  Label:           — sub-label, title case + colon
  - item           — list item under a label or section
  # Subsection:    — named subsection; use only when a section has three or
  - item             more named groups; one or two groups use plain Label: instead
  <note>           — authoring instruction or hint; remove entirely when
                     generating GPT instructions

SPACING RULES:
  2 blank lines before every SECTION NAME
  No blank line between a section heading and its first content
  1 blank line before each # subsection
  No blank line between a Label: or # Subsection: and its dash items
  1 blank line between plain Labels: within the same section

CHARACTER COUNT CHECK:
ChatGPT enforces a hard limit on GPT instructions. Keep the final file
under 7,900 characters after deleting both template blocks and all <...> content.

To check: cat instructions.txt | wc -c

If over 7,900 characters: rewrite and compress the prose to say the same
thing in fewer words. Preserve the intelligence, meaning, and intent of
every section. Do not drop sections or weaken constraints.
If compression alone cannot get under 7,900 characters, move the largest
section(s) to knowledge-<section_name>.txt and replace the section content
in instructions.txt with: "Refer to knowledge-<section_name>.txt for
instructions."
